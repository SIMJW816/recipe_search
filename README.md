# 🍲 시맨틱 레시피 검색기 (Semantic Recipe Search System)

본 프로젝트는 단순 키워드 매칭의 한계를 극복하고, 자연어에 담긴 "의미론적(Semantic)" 문맥을 이해하여 가장 적합한 레시피를 찾아내는 AI 검색 애플리케이션입니다.

## 📌 주요 핵심 기능 (Core Features)
1. **의도 추출 및 가상 문서 생성 (HyDE)**: `gpt-4.1-mini`를 사용하여 사용자의 질문에서 검색 의도(시간대, 식단 목적, 맛)를 논리적으로 분리하고 가상의 레시피 문장을 생성합니다.
2. **시맨틱 매칭 (Semantic Search)**: `text-embedding-3-small` 모델을 사용하여 자연어를 고차원 벡터로 변환, 코사인 유사도 연산을 통해 의미가 가장 가까운 레시피를 반환합니다.
3. **하이브리드 필터링 (Hybrid Filtering)**: 벡터 검색의 한계점(예: 아침과 저녁의 벡터가 가깝게 나오는 현상)을 보완하기 위해, 분리된 의도를 바탕으로 알고리즘 페널티(가중치 조절)를 부여하여 정확도를 극대화했습니다.

## 🎯 과제 요구사항 충족 및 확장 기능 요약
- **[요구사항 A] 임베딩 캐싱**: API 중복 호출을 막기 위해 100개의 방대한 데이터셋 임베딩 결과를 `.pkl`로 로컬에 캐싱합니다.
- **[요구사항 B] 시맨틱 검색**: 텍스트 선형화 기법(Enriched Embedding)을 적용하여 벡터 매칭 효율을 높였습니다.
- **[요구사항 C] 코사인 거리 구현**: 수학적 공식($Cosine\ Similarity = \frac{A \cdot B}{\|A\| \|B\|}$)에 기반한 계산 함수를 직접 구현하였고 Scipy와 오차가 없음을 화면 하단에 검증 패널로 증명했습니다.
- **[확장 기능 1] UI 구현**: Streamlit과 Expander를 활용하여 검색, 로딩 상태, 상세 레시피를 직관적으로 제공합니다.
- **[확장 기능 2] Enriched 임베딩**: 단순 요리명이 아닌 "요리명, 카테고리, 특징, 재료"를 하나의 문맥으로 묶어 임베딩 품질을 향상시켰습니다.
- **[확장 기능 3] 키워드 검색 대비 비교 분석**: 화면 하단 탭을 통해 기존의 원시적인 단어 횟수 매칭 검색(Keyword Search)과 시맨틱 검색 결과를 정량적으로 비교 분석합니다.
