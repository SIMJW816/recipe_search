# 🍲 시맨틱 레시피 검색기 (Semantic Recipe Search System)

본 프로젝트는 단순 키워드 매칭의 한계를 극복하고, 자연어에 담긴 **"의미론적(Semantic)" 문맥**을 이해하여 사용자에게 가장 적합한 요리법을 추천하는 AI 기반 검색 애플리케이션입니다. 

수업에서 학습한 **"검색과 분류는 같은 것이다"**라는 통찰을 코드로 구현하였으며, 임베딩 벡터 공간에서의 유사도 탐색과 실무적인 하이브리드 필터링 로직을 결합했습니다.

---

## 1. 프로젝트 소개 및 아키텍처 설명

본 시스템은 검색의 정확도를 극대화하기 위해 다음과 같은 3단계 파이프라인을 거칩니다.

### 🏗️ 시스템 아키텍처
1.  **의도 분석 및 가상 문서 생성 (LLM: gpt-4.1-mini)**
    -   사용자의 자연어 질의에서 '맛', '재료', '시간대', '식단 목적'을 논리적으로 분리합니다.
    -   **HyDE (Hypothetical Document Embeddings)** 기법 적용: 검색어와 매칭율이 높은 '가상의 레시피 묘사 문장'을 생성하여 벡터 매칭의 정교함을 높입니다.
2.  **의미론적 벡터 매칭 (Embedding: text-embedding-3-small)**
    -   OpenAI의 최신 임베딩 모델을 사용하여 텍스트를 고차원 벡터로 변환합니다.
    -   수학적 공식에 기반해 직접 구현한 **코사인 유사도(Cosine Similarity)**를 사용하여 벡터 공간상에서 가장 가까운 이웃 레시피를 탐색합니다.
3.  **하이브리드 페널티 시스템 (Rule-based Filtering)**
    -   벡터 공간의 한계(예: '아침'과 '저녁' 메뉴의 유사 문맥 혼동)를 보완하기 위해, 분석된 필터링 조건이 데이터의 속성과 충돌할 경우 알고리즘적으로 유사도 점수를 차감(Penalty)하거나 가점(Bonus)을 주어 최종 순위를 보정합니다.

## 2. 실행 방법 (Getting Started)

### 🛠️ 설치 및 환경 구축
1.  저장소를 클론(Clone)하거나 다운로드하여 해당 폴더로 이동합니다.
2.  터미널에서 아래 명령어를 실행하여 필수 패키지를 설치합니다.
    ```bash
    pip install -r requirements.txt
    ```

### 🔐 환경변수 설정
1.  프로젝트 최상단 폴더(`app.py`가 있는 위치)에 `.env` 파일을 생성합니다.
2.  OpenAI API 키를 아래 형식으로 입력합니다.
    ```env
    OPENAI_API_KEY=your_api_key_here
    ```
    *(주의: 보안을 위해 `.env` 파일은 GitHub 등 공용 저장소에 업로드되지 않도록 `.gitignore`에 등록되어 있습니다.)*

### ▶️ 애플리케이션 실행
터미널에서 아래 명령어를 입력하여 Streamlit 웹 서버를 실행합니다.
```bash
streamlit run app.py
```
### 3. 사용한 데이터셋 출처 및 설명

* **데이터셋 출처**: 본 과제의 시맨틱 검색 성능 및 하이브리드 필터링 로직을 정밀하게 테스트하기 위해, LLM의 보조를 받아 직접 기획하고 큐레이션하여 구축한 **커스텀 레시피 데이터셋**입니다. (외부 오픈 데이터셋 대신 프로젝트 목적에 완벽히 맞추어 설계했습니다.)
* **데이터 규모**: 총 100개의 고품질 레시피 데이터 (한식, 양식, 일식, 중식, 아시안, 분식, 야식, 다이어트식 등 포괄)
* **데이터 구성 필드**:
    * `요리명`, `맛의 특징`, `주요 재료`
    * `카테고리`: 한식/양식 등의 기본 분류 외에도, 논리적 제어를 위한 **시간대**(아침/점심/저녁/야식) 및 **식단 목적**(고단백/다이어트/해장 등) 메타데이터 포함
    * `상세 조리법 (Recipe)`: 검색 후 결과 창에서 즉시 요리가 가능하도록 제공되는 단계별(Step-by-step) 조리 가이드
* **구조적 특징 (Enriched Embedding)**: 개별 속성을 단순히 나열하지 않고, `요리명: ~, 카테고리: ~, 맛의 특징: ~, 재료: ~` 형태의 자연스러운 하나의 문맥으로 묶는 **풍부한 임베딩(Enriched Embedding)** 전처리를 적용했습니다. 이를 통해 AI 모델이 레시피의 복합적인 의미를 훨씬 뚜렷하게 인식하도록 설계했습니다.
